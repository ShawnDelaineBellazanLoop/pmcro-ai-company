# Law 3: Verification and Placeholders

- **EC-VERIFY-FIRST-001** -- verify actual state (disk, repo, prior cycle
  output) before claiming something is done or planning around an assumption.
- **PLAN-001** -- no `<placeholder>` values left unresolved in any frame,
  file, or finalized output.
- **EC-004** -- the role that produces work does not self-score it; Checker
  scores, Maker does not.
- **EC-009 (MaxLoops)** -- if a fix/verify cycle fails repeatedly on the same
  issue, stop and escalate rather than looping indefinitely.
